# 01 - Objetivos de Aprendizagem

> **"Aprender não significa apenas adquirir conhecimento, mas desenvolver competências capazes de resolver problemas reais."**

---

## Introdução

Todo capítulo da Academia de Engenharia de Dados inicia com objetivos claros de aprendizagem.

Ao definir antecipadamente as competências que serão desenvolvidas, o estudante consegue compreender a importância do conteúdo, acompanhar sua evolução e avaliar o próprio progresso.

Este documento apresenta os objetivos específicos deste capítulo, os conhecimentos esperados e como eles se conectam aos próximos volumes da formação.

---

## Objetivo Geral

Ao concluir este capítulo, o estudante compreenderá a origem, a evolução e a importância da Engenharia de Dados nas organizações modernas, sendo capaz de explicar seu papel dentro do ecossistema de dados e reconhecer os principais desafios enfrentados pelos profissionais da área.

---

## Objetivos Específicos

Ao final deste capítulo, você deverá ser capaz de:

### Compreender a evolução dos dados

- Explicar como os sistemas de informação evoluíram nas últimas décadas.
- Identificar os principais marcos tecnológicos da área de dados.
- Relacionar a evolução tecnológica com as necessidades do mercado.

---

### Entender o nascimento da Engenharia de Dados

- Explicar por que surgiu a Engenharia de Dados.
- Identificar os problemas que essa disciplina resolve.
- Compreender sua evolução a partir de Business Intelligence e Big Data.

---

### Conhecer o papel do Engenheiro de Dados

- Identificar as responsabilidades do profissional.
- Diferenciar atribuições técnicas e estratégicas.
- Compreender a interação com outras equipes.

---

### Compreender o ecossistema de dados

Ao finalizar este capítulo você deverá conseguir diferenciar claramente:

- Engenharia de Dados
- Business Intelligence
- Analytics
- Ciência de Dados
- Machine Learning
- Inteligência Artificial
- DataOps
- MLOps

---

### Entender o ciclo de vida dos dados

Você deverá ser capaz de explicar cada etapa do fluxo de dados:

- geração;
- ingestão;
- armazenamento;
- transformação;
- enriquecimento;
- disponibilização;
- consumo;
- monitoramento.

---

### Desenvolver visão arquitetural

Ao concluir este capítulo você deverá reconhecer os principais componentes de uma plataforma moderna de dados e compreender como eles se relacionam.

Essa visão será aprofundada ao longo dos próximos volumes.

---

## Competências Desenvolvidas

As competências deste capítulo estão organizadas em quatro dimensões.

### 1. Conhecimento Conceitual

Você compreenderá:

- os conceitos fundamentais da Engenharia de Dados;
- a terminologia utilizada pelo mercado;
- as principais arquiteturas modernas;
- a evolução da área.

---

### 2. Pensamento Sistêmico

Você desenvolverá a capacidade de:

- visualizar o fluxo completo dos dados;
- compreender a integração entre diferentes tecnologias;
- identificar dependências entre componentes de uma plataforma.

---

### 3. Visão de Negócio

Você compreenderá que Engenharia de Dados não existe apenas para armazenar informações.

Seu principal objetivo é entregar dados confiáveis para apoiar:

- decisões estratégicas;
- análises;
- produtos digitais;
- Inteligência Artificial;
- automação de processos.

---

### 4. Desenvolvimento Profissional

Ao concluir este capítulo você compreenderá:

- como evoluir na carreira;
- quais competências são exigidas pelo mercado;
- quais tecnologias serão estudadas durante a Academia.

---

## Competências Esperadas por Nível

### Ao final deste capítulo

Você deverá possuir uma visão introdutória sobre Engenharia de Dados.

Ainda não será esperado que saiba implementar pipelines ou arquiteturas.

O foco é compreender os conceitos fundamentais.

---

### Após concluir a Academia

Ao finalizar todos os volumes você será capaz de:

- construir pipelines de dados;
- desenvolver plataformas modernas de dados;
- implementar arquiteturas Lakehouse;
- trabalhar com processamento distribuído;
- desenvolver soluções utilizando Spark, PostgreSQL, Trino, Airflow, dbt e outras tecnologias estudadas;
- monitorar ambientes de produção;
- aplicar boas práticas de Engenharia de Dados.

## Critérios de Conclusão

Considere este capítulo concluído quando você conseguir responder, sem consultar o material, às seguintes perguntas:

- O que é Engenharia de Dados?
- Por que essa disciplina surgiu?
- Qual é a diferença entre Engenharia de Dados e Ciência de Dados?
- Quais problemas um Engenheiro de Dados resolve?
- Como ocorre o ciclo de vida dos dados?
- Qual é o papel da Engenharia de Dados dentro de uma empresa?
- Quais tecnologias fazem parte de uma plataforma moderna de dados?

Se conseguir responder a essas perguntas com segurança, você estará preparado para avançar para os próximos tópicos.
