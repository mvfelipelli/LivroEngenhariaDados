# Capítulo 01 — O que é Engenharia de Dados

> "Os dados são o novo petróleo" tornou-se uma frase popular, mas incompleta. Dados, por si só, não possuem valor. O valor surge quando eles são coletados, organizados, transformados, disponibilizados e utilizados para apoiar decisões e criar produtos. É exatamente nesse contexto que nasce a Engenharia de Dados.

---

## Apresentação

Bem-vindo ao primeiro capítulo da Academia de Engenharia de Dados.

Este capítulo apresenta os conceitos fundamentais da Engenharia de Dados e estabelece a base sobre a qual todo o restante da formação será construído.

Antes de aprender tecnologias como Spark, PostgreSQL, Trino ou Airflow, é essencial compreender **por que essas ferramentas existem**, **quais problemas elas resolvem** e **como elas se integram dentro de uma plataforma moderna de dados**.

Ao longo deste capítulo exploraremos a evolução histórica da área, o papel do Engenheiro de Dados nas organizações, as competências exigidas pelo mercado e a importância dessa disciplina para Analytics, Ciência de Dados, Inteligência Artificial e produtos digitais.

Este capítulo não exige qualquer conhecimento prévio sobre Engenharia de Dados.

---

## Objetivos de Aprendizagem

Ao concluir este capítulo, você será capaz de:

- compreender a origem da Engenharia de Dados;
- explicar por que essa disciplina tornou-se essencial nas organizações modernas;
- diferenciar Engenharia de Dados de Business Intelligence, Analytics, Ciência de Dados e Inteligência Artificial;
- compreender o ciclo de vida dos dados dentro de uma empresa;
- identificar as principais responsabilidades de um Engenheiro de Dados;
- conhecer as competências técnicas e comportamentais exigidas pela profissão;
- entender como a Engenharia de Dados se integra às demais áreas de tecnologia.
