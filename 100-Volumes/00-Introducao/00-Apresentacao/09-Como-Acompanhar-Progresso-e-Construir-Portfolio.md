# Como Acompanhar Progresso e Construir Portfólio

Progresso deve ser medido por capacidade demonstrada, não apenas páginas lidas. Para cada módulo, mantenha notas de decisão, exercícios respondidos, laboratório executado e uma explicação curta do resultado.

Um portfólio técnico útil apresenta problema, contexto, arquitetura, código, testes, métricas, limitações e próximos passos. Remova segredos e dados pessoais; forneça dados sintéticos e comandos reproduzíveis.

| Evidência | Pergunta respondida |
|---|---|
| README | O que foi construído? |
| Diagrama | Como as partes se relacionam? |
| Testes | Como sabemos que funciona? |
| Runbook | Como operar e recuperar? |
| Decisão | Por que esta solução? |

Commits pequenos e mensagens claras tornam a evolução visível.
