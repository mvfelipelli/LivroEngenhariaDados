# Marcos, Entregas e Dependências entre Volumes

Cada volume adiciona uma capacidade e revisita decisões anteriores.

| Etapa | Entrega do projeto |
|---|---|
| Fundamentos | contratos e arquitetura conceitual |
| Linux/Git | ambiente e versionamento |
| SQL/Modelagem | modelo e consultas reconciliadas |
| Python | ingestão e validação automatizadas |
| Spark | processamento distribuído |
| PostgreSQL | persistência e operação relacional |
| Lakehouse/Trino | tabelas abertas e consulta federada |
| Airflow | orquestração e backfill |
| Qualidade/Observabilidade | SLOs e resposta |
| Projeto final | plataforma integrada |

Uma entrega só é incorporada quando possui documentação, teste e critério de aceite. O roadmap pode mudar, mas dependências precisam permanecer explícitas.
