# Resumo

- Classes agrupam estado e comportamento coesos.
- Instâncias devem nascer válidas e transições preservar invariantes.
- Propriedades controlam uma API de atributo.
- Composição reduz acoplamento; herança exige substituição segura.
- MRO define a busca em herança múltipla.
- Dataclasses modelam registros; frozen e slots refinam o contrato.
- Type hints são graduais e geralmente não validam runtime.
- Protocols descrevem interfaces estruturalmente.
- Generics preservam relações entre tipos.
- `Any` e `cast` devem ficar restritos a fronteiras comprovadas.
