# Resumo

- CPython compila código para bytecode e o executa em uma máquina virtual.
- O processo expõe argumentos, ambiente, streams e código de saída.
- A versão suportada deve ser declarada e testada.
- `venv` isola pacotes por projeto, mas não todo o sistema operacional.
- `pyproject.toml` centraliza metadados e configuração.
- Lock e hashes tornam instalações mais determinísticas.
- REPL, script, módulo, pacote e CLI atendem usos distintos.
- Formatação, lint, tipos e testes formam um ciclo complementar.
- Configuração não é código; segredo não é configuração versionável.
- Uma execução profissional começa por um contrato reproduzível.

O próximo módulo aplica essa base a tipos, expressões, decisões, repetições e coleções.
