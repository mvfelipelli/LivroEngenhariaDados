# 01 — Objetivos

> [!info]
> Todo projeto de Engenharia de Dados começa com uma compreensão clara do caminho que os dados percorrem dentro de uma organização. Este módulo apresenta essa jornada de ponta a ponta.

---

# Objetivo Geral

Compreender o **Ciclo de Vida dos Dados**, identificando cada uma de suas etapas, seus objetivos, desafios e a forma como elas se conectam para transformar dados brutos em informações úteis para o negócio.

Ao final deste módulo, o leitor deverá ser capaz de enxergar qualquer ambiente de dados sob a perspectiva de um fluxo contínuo, independentemente das tecnologias utilizadas.

---

# Objetivos Específicos

Ao concluir este módulo, você será capaz de:

## Compreender os fundamentos

- explicar o que é o Ciclo de Vida dos Dados;
- entender por que esse conceito é central na Engenharia de Dados;
- reconhecer que os dados possuem um ciclo semelhante ao de qualquer outro ativo corporativo.

---

## Identificar as etapas do ciclo

Você será capaz de reconhecer e descrever cada uma das etapas:

- geração;
- coleta;
- ingestão;
- armazenamento;
- processamento;
- consumo;
- compartilhamento;
- arquivamento;
- descarte seguro.

Além disso, compreenderá como essas etapas se relacionam entre si.

---

## Entender os papéis envolvidos

Ao longo do ciclo, diferentes profissionais atuam sobre os dados.

Você compreenderá o papel de:

- Engenheiro de Dados;
- Cientista de Dados;
- Analista de Dados;
- Desenvolvedor de Software;
- Administrador de Banco de Dados (DBA);
- Arquiteto de Dados;
- Gestores de Negócio;
- Equipes de Governança e Segurança.

---

## Relacionar o ciclo às arquiteturas modernas

Ao finalizar este módulo você compreenderá por que arquiteturas como:

- Data Warehouse;
- Data Lake;
- Lakehouse;
- Data Mesh;
- Data Fabric;

existem para dar suporte ao ciclo completo dos dados.

---

## Identificar riscos

Você aprenderá a reconhecer problemas comuns, como:

- perda de dados;
- duplicidade;
- inconsistências;
- baixa qualidade;
- processamento inadequado;
- ausência de governança;
- falhas de segurança;
- retenção inadequada;
- descarte incorreto.

---

## Relacionar teoria e prática

Durante todo o módulo, os conceitos serão aplicados ao projeto integrador da **DataRetail S.A.**, permitindo visualizar como empresas reais lidam com o fluxo de dados em suas operações diárias.

---

# Competências Desenvolvidas

Ao concluir este módulo, espera-se que você desenvolva as seguintes competências.

## Competências Técnicas

- compreender o fluxo completo dos dados;
- identificar pontos críticos de um pipeline;
- reconhecer responsabilidades de cada etapa;
- interpretar arquiteturas de dados sob a ótica do ciclo de vida;
- compreender como diferentes tecnologias participam da jornada dos dados.

---

## Competências Analíticas

- analisar fluxos de informação;
- identificar gargalos;
- reconhecer oportunidades de melhoria;
- avaliar riscos relacionados aos dados;
- compreender impactos das decisões arquiteturais.

---

## Competências Profissionais

Ao longo deste módulo você desenvolverá uma visão sistêmica da Engenharia de Dados, habilidade essencial para profissionais que atuarão em projetos de:

- Analytics;
- Business Intelligence;
- Machine Learning;
- Inteligência Artificial;
- Big Data;
- Governança;
- Plataformas de Dados.

---

# O que você NÃO aprenderá neste módulo

> [!note]
> Este módulo é conceitual. O objetivo é construir uma base sólida antes da introdução das tecnologias específicas.

Não serão abordados detalhes de implementação utilizando ferramentas como:

- PostgreSQL;
- Apache Spark;
- Kafka;
- Airflow;
- Hive;
- Trino;
- dbt;
- Iceberg;
- Delta Lake.

Essas tecnologias serão estudadas em profundidade nos módulos posteriores, quando o leitor já compreender o papel que desempenham dentro do ciclo de vida dos dados.

---

# Ao concluir este módulo...

Você será capaz de observar qualquer ambiente de dados — independentemente da tecnologia utilizada — e identificar:

- onde os dados são produzidos;
- como são coletados;
- como chegam à plataforma de dados;
- onde são armazenados;
- como são transformados;
- quem consome essas informações;
- quando devem ser arquivados;
- quando precisam ser descartados.

Essa visão sistêmica será utilizada continuamente ao longo de toda a Formação em Engenharia de Dados.
