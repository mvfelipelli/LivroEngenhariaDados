# Exercícios

## Revisão

1. Compare shared repository e fork workflow.
2. Diferencie review, status check e ruleset.
3. Compare merge, squash, rebase e cherry-pick.
4. Explique CODEOWNERS e merge queue.

## Interpretação

5. PR foi aprovada, recebeu novo commit e foi integrada. Qual risco?
6. Workflow de fork acessa segredo de produção. Diagnostique.
7. Duas PRs verdes quebram quando integradas juntas. Proponha controle.
8. Admin faz bypass semanalmente. O que isso revela?

## Aplicação

9. Escreva template de PR para migration.
10. Defina ruleset da branch principal.
11. Crie CODEOWNERS para SQL, infra e workflows.
12. Modele release ligando issue, PR, tag e deploy.

## Desafios

13. Compare trunk-based e release branches para produto regulado.
14. Modele hotfix com auditoria e baixo tempo de resposta.
15. Execute [[14-Laboratorio]] e explique o papel de cada clone.

Consulte [[13-Gabarito]] depois de justificar as respostas.
