# Gabarito

1. Shared usa branches no mesmo repo; fork separa repositório e escrita, alterando confiança.
2. Review é decisão humana; check é resultado automatizado; ruleset aplica condições.
3. Merge preserva grafo, squash consolida, rebase recria série e cherry-pick copia mudança.
4. CODEOWNERS mapeia caminhos a owners; queue valida integração sobre base atualizada.
5. Aprovação pode estar obsoleta; exija dismissal após novos commits.
6. Código não confiável pode exfiltrar segredo; remova segredo e redesenhe evento/permissões.
7. Use merge queue ou teste da combinação com base mais recente.
8. Regra não corresponde ao processo ou cultura; investigue causa, restrinja e audite bypass.
9. Inclua contexto, schema, compatibilidade, dados, teste, rollout e rollback.
10. PR, revisão, checks, conversas, bloqueio de force/delete e bypass restrito.
11. Use padrões específicos e owners múltiplos para workflows críticos.
12. Issue define demanda; PR implementa; tag identifica; release comunica; deployment prova ambiente.
13. Trunk reduz divergência; releases paralelas atendem suporte, com maior custo de backport.
14. PR pequena, review síncrona, checks essenciais, deploy controlado e post-review de exceção.
15. Central representa upstream; Alice propõe; Bob revisa, integra e publica `main`.
