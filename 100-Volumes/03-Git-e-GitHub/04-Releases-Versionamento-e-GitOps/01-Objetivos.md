# Objetivos

Ao concluir o módulo, o leitor será capaz de:

- distinguir versão, tag, release, artefato, deploy e promoção;
- aplicar SemVer a uma API pública explícita;
- projetar releases reproduzíveis com proveniência e rollback;
- explicar os quatro princípios de GitOps;
- modelar reconciliação, detecção de drift e segregação de ambientes;
- avaliar segurança, governança e métricas do fluxo;
- implementar e testar um controlador GitOps mínimo.

O percurso conecta [[03-GitHub-Actions-e-CI-CD/README|CI/CD]] à operação declarativa sem confundir entrega com implantação.
