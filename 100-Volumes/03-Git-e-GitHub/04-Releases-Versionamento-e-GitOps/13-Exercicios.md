# Exercícios

## Revisão

1. Diferencie versão, tag, release, artefato e deploy.
2. Liste os quatro princípios de GitOps.

## Interpretação

3. Uma API `2.3.4` adiciona campo opcional. Qual versão deve seguir?
4. Explique por que reconstruir a imagem para produção é arriscado.
5. Um operador muda réplicas de três para cinco fora do Git. Caracterize o evento.

## Aplicação

6. Proponha gates para promover um pipeline da DataRetail à produção.
7. Modele uma política de tags e releases assinadas.
8. Defina tratamento de segredos para um repositório GitOps.

## Desafio

9. Projete um fluxo canary com rollback automático e critérios mensuráveis.
10. Descreva um procedimento break-glass que preserve auditoria e restaure a fonte autoritativa.
